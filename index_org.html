Sequential Presents Pi

Hello Artblocks folks, thanks for checking us out! We are WaveWorks, and are thrilled to introduce our latest project: “Sequential Presents Pi”

We have been working hard for the past few months to revive a project we created nearly 15 years ago to visually represent sequential data. The original purpose of our program was analyzing genome sequences, but when testing our project, we would often use the digits of Pi. While we were never able to get the product off the ground, we never forgot the beauty of Pi.

When we learned of ArtBlocks, we realized the artistic potential of the project and immediately set about re-creating a prototype.

Our fully functional prototype uses a random hash to generate attributes that are then used to create the animation.

It is both dimensionally agnostic and deterministic. We often save hashes of our favorite combinations of attributes.

Sequential Mechanics

Sequential is a histogram taken to the extreme.

First, let's build a histogram using the digits of Pi. In the video below, you can see how the histogram is assembled. The digits are arranged from 0 to 9 and left to right. As we progress through the digits of Pi, we add a square to the appropriate column.

Now that we have a histogram in place, let's add Sequential to it. How is that done? By using a sequence of colors to reveal the order in which the digits of Pi arrived. For example, what if we started with a color gradient from green to black? Then divide it into 10 distinct color steps? As the shapes are added to the histogram, we cycle through the gradient sequence starting with green and moving towards black. When all the color steps have been used, we restart the gradient sequence from the beginning.

The final ingredient needed for Sequential is what we call flow. By stepping each shape through the color sequence, we begin to see motion.

Sequential History and Originality

Due to recent events, we understand you are concerned about originality. Our first Sequential prototype was written in Processing 1.0. In fact, our old posts can still be found on the processing.org forum. For example, here's a post by 4dplane from 2006 working on a color blending issue related to Sequential.

https://processing.org/discourse/beta/num_1143770241.html

We do not have our original Processing code, because we eventually migrated to Java, so we started Sequential Presents Pi from scratch in p5js for ArtBlocks.

We also have a wayback machine archive from 2014 describing our project.

Sequential on WayBack Machine

All Sequential code is 100% original except for the Pi digit generator, which is under GNU Free Documentation License and can be found here: Spigot Algorithm

Currently, Pi is generated with the Spigot Algorithm to save on code size, and there is the opportunity for Sequential to produce infinite none repeating art. In reality, computer hardware limits the Spigot Algorithm by the fact that each new digit takes more processing time, so we are exploring other options for accessing the digits of Pi. One possibility we are considering, is to hard code 5k-10k digits of Pi and then loop Sequential back to the beginning once there are no new digits. An extension to this idea, is to store and access additional digits of Pi on the ETH blockchain. By doing this, we would be able to expand Sequential's exploration of Pi by allowing more digits to be added to the ETH blockchain at anytime and by anyone.
